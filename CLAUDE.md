# CLAUDE.md — 扶摇阁

## 开发工作流

### Bug 修复（例外）

修 bug 可以直接改代码，跳过文档更新步骤，但修复完成后仍需编译安装验证、提交 GitHub。

### 非 Bug 改动（迭代、新增功能、删减功能等）

所有非 bug 修复的改动，**必须先改文档、再改代码**，按以下阶段执行：

**阶段一：更新设计文档（先于任何代码改动）**

**⚠️ 关卡 0（必须）**：任何 UI 改动或设计方案，在写文档之前，**必须先向用户展示设计草案并获得明确审批**。用户确认方向后方可进入文档阶段。禁止跳过审批直接动手。

1. **UI_DESIGN.md** — 如果改动涉及 UI，先更新设计稿，确认布局、层级、颜色、字体、间距；不涉及 UI 则跳过
2. **PRD.md** — 更新产品需求文档，明确功能范围、业务规则、数据模型变更
3. **TASKS.md** — 更新任务列表，将改动拆成独立、可验证的子任务，每个 task 有明确的完成标准

**阶段二：实现代码**

4. **拆分 Task** — 用 TaskCreate 将 TASKS.md 中的子任务录入系统
5. **按 Task 执行** — 按 task 顺序逐项实现，完成一项标记一项

**阶段三：验证与提交**

6. **编译安装** — 代码改动完成后，编译通过并安装到设备
7. **提交 GitHub** — 验证无问题后，将改动提交并推送到 GitHub
