# 扶摇阁 — UI 高级感重设计方案

> 基于 PRD v15 章节，目标：Apple 级别精致感 × 金融专业信任感

---

## 一、设计理念

### 核心关键词

**克制 · 层次 · 流动 · 金色点缀**

- **克制**：减少颜色数量，信息层级分明，留白充足
- **层次**：白色卡片 + 阴影叠在暖灰背景上，深度感清晰
- **流动**：微动画（数字过渡、涟漪、fade+slide 页面切换）
- **金色点缀**：少量金色（CTA / 选中态 / 分割线）暗示财富属性，不喧宾夺主

### 对标参考

| 维度 | 参考 |
|------|------|
| 卡片质感 | Apple Wallet / 健康 App |
| 信息层级 | Robinhood / 富途牛牛 |
| 色彩控制 | Notion（少色、大量留白） |
| 字体选择 | 高端杂志排版（Vogue / Monocle） |

## 二、字体系统

| 角色 | 字体 | 字重 | 字号 | 行高 |
|------|------|------|------|------|
| Hero 数字（总资产） | Bodoni Moda | 700 | 36sp | 44sp |
| 大标题 | Bodoni Moda | 600 | 24sp | 32sp |
| 卡片标题 | Jost | 600 | 16sp | 24sp |
| 正文 | Jost | 400 | 14sp | 20sp |
| 辅助文本 / 标签 | Jost | 400 | 12sp | 16sp |
| 数字（金额、百分比） | Jost | 500-600 | 16-22sp | — |

> **为什么不用系统默认字体**：Bodoni Moda 的优雅衬线 + Jost 的几何现代感形成「经典 × 当代」对比，传递高级品牌感。这两个字体通过 Google Fonts 分发的 `downloadableFonts` 集成，无需打包字体文件。

### 字重使用规则

```
ExtraBold (700) → 仅 Hero 数字（总资产、今日收益主数字）
Bold (600)     → 分类卡片金额、页面标题
Medium (500)   → 次要数字、按钮文字
Regular (400)  → 正文、标签、辅助信息
```

## 六、全 App 连续滑动体系（v3 核心架构）

### 6.0 设计目标

**消灭一切 Tab 点击切换，改为连续滑动。** 用户从首页开始向右滑，可以无中断地穿越所有页面。

底部导航不再触发页面跳转，而是**快速定位到 Pager 中的锚点页面**。

### 6.1 页面序列（扁平化 HorizontalPager）

```
Page  0:  首页（净资产仪表盘）
Page  1:  投资 · 股票
Page  2:  投资 · 基金
Page  3:  投资 · 黄金
Page  4:  存款
Page  5:  现金
Page  6:  信用
Page  7:  收益
Page  8:  设置
```

**总共 9 页，一个 `HorizontalPager`，userScrollEnabled = true。**

> v18 合并存款页：持有中/已到期不再分页，已到期存单 inline 显示（半透明 + badge）。

### 6.2 底部导航 → Pager 锚点映射

```
底部:  [首页]    [持仓]    [收益]    [设置]
        ↓         ↓         ↓         ↓
Page:   0         1         7         8
```

- 点「首页」→ `animateScrollToPage(0)`
- 点「持仓」→ `animateScrollToPage(1)`（投资·股票，持仓段第一页）
- 点「收益」→ `animateScrollToPage(7)`
- 点「设置」→ `animateScrollToPage(8)`
- **底部导航选中态跟随 `currentPage` 所在区间变化**

### 6.3 滑动规则

| 规则 | 说明 |
|------|------|
| 全程可滑 | `userScrollEnabled = true`，惯性滑动 |
| 无循环 | 第 0 页不能左滑，第 8 页不能右滑 |
| 二级优先 | 同一分类内的子页先滑完（股票→基金→黄金），再进入下一个分类（存款） |
| 底部同步 | 滑到 page 0 → 首页高亮；滑到 page 1-6 → 持仓高亮；page 7 → 收益；page 8 → 设置 |
| 首页资产网格 | 点击网格里的「现金」「存款」等，不再跳 page，而是 `animateScrollToPage(目标页)` |

### 6.3 Category 快速跳转导航条（v4 新增）

**问题**：10 页扁平 Pager 中，从股票(page 1) 到存款(page 4) 需滑 3 次。仅靠滑动无法快速跨越分类。

**方案**：在 Pager 上方（SectionIndicator 上方）新增一条可点击的分类导航条，**v17 增加分类汇总行**：

```
┌──────────────────────────────────────┐
│  [投资]    [存款]    [现金]   [信用] │  ← CategoryNavStrip
│   ●                                │     当前分类高亮+金色下划线
│                                      │
│  投资总计  ¥423,000  今日 +¥2,500   │  ← v17 新增：当前分类汇总
└──────────────────────────────────────┘
```

**规格**：
- 仅在 page 1-6 显示（投资段+存款+现金+信用）
- 4 个分类 chip：投资 / 存款 / 现金 / 信用
- 当前分类 = 金色文字 + 2dp 底部指示线，其他 = 灰色文字
- 点击 → `animateScrollToPage(该分类第一页)`
- 导航条背景白色卡片，16dp 圆角，左右 margin 16dp
- 映射：投资→page 1, 存款→page 4, 现金→page 5, 信用→page 6

**v17 分类汇总行规格**：
- 位于 chip 行下方，一条细分割线隔开
- 左侧：分类名 + 总计（如「投资总计 ¥423,000」），14sp Jost 500
- 右侧：今日变动（涨跌色），14sp Jost 600，如「+¥2,500」
- 随当前 page 所在分类动态切换：

| 当前分类 | 汇总行内容 |
|----------|-----------|
| 投资 (page 1-3) | 投资总计 ¥423,000 / 今日 +¥2,500 |
| 存款 (page 4) | 存款总计 ¥512,350 / 今日利息 +¥45.20 |
| 现金 (page 5) | 现金余额 ¥45,000（今日不适用） |
| 信用 (page 6) | 应收净额 -¥30,000（今日不适用） |

**指示器规格（v17 更新）：**

- 子项名称（股票/基金/黄金、持有中/已到期）**可点击**，点击 → `animateScrollToPage(目标页)`
- 子类汇总行：市值 14sp Jost 600 `onSurface`，今日涨跌 13sp Jost 500 涨跌色

### 2×2 资产网格卡片规格

每张卡片 = 顶部色条（4dp）+ 两行内容，半宽紧凑布局：

```
┌────────────────────┐
│ ════════════════   │  ← 顶部色条 4dp, 资产类型色
│  投资    ¥420,000   │  ← 行1: 名称(14sp Bold) + 总价值(16sp Bold)
│  +820 今日         │  ← 行2: 今日收益(13sp, 涨跌色)
└────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 卡片 | `ElevatedCard`, 14dp 圆角, elevation=2dp |
| 卡片宽度 | `Modifier.weight(1f)`，两列等宽 |
| 顶部色条 | 4dp 高, 资产类型色, 卡片顶部圆角跟随 |
| 列间距 | 10dp |
| 行间距 | 10dp |
| 卡片内边距 | 10dp 水平, 8dp 垂直（色条下方） |
| 名称 | 14sp, SemiBold, onSurface |
| 总价值 | 16sp, Bold, onSurface（右对齐） |
| 今日收益 | 13sp, SemiBold, 涨跌色（无收益则不显示） |
| 最小高度 | 72dp（保证四张卡片等高） |

```
Row 1:  投资（左上）    存款（右上）
Row 2:  现金（左下）    信用（右下）
```

按总价值绝对值降序：投资 > 存款 > 现金 > 信用（应收净额）。

### 交互

| 操作 | 行为 |
|------|------|
| **点卡片** | `animateScrollToPage(目标页)` |
| **点总资产数字** | 无操作（纯展示） |
| **下拉刷新** | 拉取实时价格 + 重算所有数据 |
| **长按卡片** | 弹出快捷菜单（投资→添加持仓，存款→添加存款，现金→入金/出金） |
| **下拉刷新** | 拉取实时价格 + 重算所有数据 |
| **右滑** | 自然进入投资·股票（连续 Pager） |

### 我的额外优化建议

| # | 优化点 | 说明 |
|---|--------|------|
| 1 | **数字动画** | 价格刷新后数字用 `animateFloatAsState` 过渡，500ms easeOut，不是瞬间跳变 |
| 2 | **骨架屏** | 启动时价格还没拉回来，卡片显示 shimmer 占位，避免空白闪烁 |
| 3 | **长按快捷操作** | 长按投资卡片 → 「添加持仓」；长按存款 → 「添加存款」；减少导航步数 |
| 4 | ~~总资产点按切换精度~~ | 已废弃：全部使用完整数字，不用"万"缩写 |
| 5 | **今日收益区域下拉展开** | 下拉总资产区域 → 展开显示近 7 天每日收益小列表（不需要跳到收益页） |
| 6 | **负债卡片视觉弱化** | 应收/负债卡片用更低的 elevation + 更淡的左边条，不跟资产卡片抢注意力 |
| 7 | **触觉反馈** | 卡片点击加 `hapticFeedback`，手感更扎实 |

### 8.5 符号统一规则（全局适用）

**所有视图（日/周/月/年）严格统一：**

```
正值 → + 号 → 红色 FinancialColors.up     (#C0392B)
负值 → - 号 → 绿色 FinancialColors.down   (#27AE60)
零值 → "0"  → 灰色 onSurfaceVariant
```

- `formatMoney(value)` 本身不处理正数符号，只做千分位 + 两位小数；负数自带 `-`
- 调用方统一处理：`if (value >= 0) "+${formatMoney(value)}" else formatMoney(value)`
- `CalendarView.formatCompact()` 需要修复：当前对负值取了 `abs()` 导致丢失 `-` 号
- 所有视图的格式化逻辑全部走 `FormatUtils.formatSignedChange()` 或等价逻辑

## 十四、持仓分组重设计（v17 — 2026-05-23 设计稿）

### 14.0 设计目标

**问题**：当前持仓列表是扁平列表，同一平台的存款、同一只股票的多次买入分散展示，信息碎片化。

**方案**：
- **存款**：同一银行/平台（`bank` 字段相同）的存单合并为一个银行组卡片，点击进入银行详情看到具体存单
- **股票/基金**：同一代码（`symbol` 相同）的持仓合并为一个标的卡片，卡片内展示买入记录
- **黄金**：保持扁平（通常每个用户只有 1-2 笔黄金）

## 十六、v19.1 UI 细节优化（2026-05-23）

### 16.1 持仓卡片交替底色

**问题**：连续的白底卡片在滚动时缺乏视觉分界，用户难以区分相邻卡片。

**方案**：为列表中的连续卡片添加极淡的交替底色，仅用于视觉分隔，不抢夺内容注意力。

**规格**：
| 属性 | 值 |
|------|-----|
| 偶数位卡片 | `surface`（正常白色） |
| 奇数位卡片 | `surfaceVariant.copy(alpha = 0.3f)`（极淡暖灰） |
| 过渡 | 无动画（直接渲染），避免滚动时的"跳跃感" |
| 适用范围 | HoldingGroupCard、HoldingCard、BankGroupCard 列表 |
| 不适用范围 | 展开的买入记录卡片（buy records） |

**原则**：色差足够在滚动时分辨卡片边界，但静止状态下几乎注意不到（`alpha = 0.3f` 非常淡）。

### 16.2 收益数字溢出处理

**问题**：在周视图/月视图的网格卡片中，金额文字过长时会被裁剪，用户可能只看到小数点或部分数字。

**方案**：所有网格卡片中的金额 Text 统一添加溢出处理。

**规则**：
- 所有 `maxLines = 1` 的金额 Text 必须添加 `overflow = TextOverflow.Ellipsis, softWrap = false`
- 窄卡片（周网格、月网格）中的金额字体从 15sp → 14sp，给更多空间
- Grid 中的数值在内容过长时自动省略号截断，绝不显示残缺数字

**影响范围**：
- `WeekEarningsView` 周块卡片金额
- `MonthEarningsView` 月份网格金额
- `YearEarningsView` 年度卡片金额
- `DayEarningsView` 月度汇总金额

## 十七、v20 信息密度与视觉层次优化

> **目标**：解决数据显示不全（省略号）、选中态丑陋、存单卡片难以区分的三大体验问题

### 17.1 颜色即语义 —— 去掉 +/- 符号

**问题**：当前所有涨跌金额使用 `+value` / `-value` 格式，在窄卡片中浪费 1 个字符空间。

**方案**：涨跌信息**仅通过颜色传达**，数字本身不显示符号。

**规则**：
- `isUp = true` → 红色文字 `FinancialColors.up`，显示绝对值
- `isUp = false` → 绿色文字 `FinancialColors.down`，显示绝对值
- `0` 值 → 使用 `onSurfaceVariant` 灰色，不区分涨跌
- 废除 `formatSignedChange()` 在此场景的使用，改用 `formatMoney()` + 绝对值

**影响范围**：
- 日/周/月/年 收益视图的所有金额 Text
- 持仓盈亏、今日利息等所有涨跌金额
- 汇总卡片的合计金额

**代码实现**：
```kotlin
// Before
val text = if (isUp) "+${formatMoney(value)}" else formatMoney(value)
// After
val displayValue = value.abs()
val text = formatMoney(displayValue)
```

### 17.2 自适应数字 —— 消除省略号

**问题**：v19.1 的 `maxLines=1 + overflow=Ellipsis` 在窄卡片（月网格 ~90dp）中频繁截断数字，用户看不到完整金额。

**方案**：采用三层自适应策略，确保数字始终完整显示。

**规则**（按优先级）：
1. **去小数**：网格卡片使用 `formatMoneyShort()`（0 位小数），节省 "XX.00" 的 3 个字符
2. **去符号**：结合 17.1，省去 +/- 符号
3. **自适应字号**：金额超过 7 位数字时字号缩小 2sp，超过 9 位时缩小 3sp
4. **允许换行**：极端情况（>10 位数字）允许 `maxLines=2`，字号再缩小

**各视图字号规格**：

| 视图 | 默认字号 | 7位+ 字号 | 9位+ 字号 | 格式 |
|------|---------|----------|----------|------|
| 月网格卡片 | 14sp | 12sp | 11sp | `formatMoneyShort` |
| 周网格卡片 | 16sp | 14sp | 12sp | `formatMoneyShort` |
| 年卡片 | 16sp | 14sp | 13sp | `formatMoneyShort` |
| 日汇总头 | 18sp | 16sp | 14sp | `formatMoney` |
| 明细面板 | 14sp | 13sp | 12sp | `formatMoney` |

### 17.3 选中态重设计 —— 金色替代灰色

**问题**：当前选中态使用 `surfaceVariant.copy(alpha = 0.45f)` 产生灰蒙蒙的脏色效果，用户反馈"太丑了"。

**方案**：所有选中态统一使用金色点缀，与设计理念一致。

**规则**：
- **卡片选中**：金色边框 `border(1.5.dp, FinancialColors.gold, shape)` + 金色标签文字
- **取消灰色背景**：选中卡片背景保持 `surface`（白色），不叠加灰色
- **未选中卡片**：无边框，靠 elevation 阴影区分
- **BankGroupCard**：移除永久 `border(0.5.dp, outlineVariant)` — 卡片间距 + elevation 足够区分

**周视图卡片实例**：
```
未选中：白色表面 + 2dp 阴影 + 常规文字
选中：  白色表面 + 2dp 阴影 + 1.5dp 金边 + 金色标签 + 金色数字
```

**存款组卡片**：
```
移除：border(0.5.dp, outlineVariant, 16.dp)  ← 这行删掉
保留：elevation=4dp 阴影区分卡片
```

### 17.4 存单卡片差异化

**问题**：银行详情页的存单卡片全部相同（蓝色左条 + 相同字号/间距），难以快速区分不同存单。

**方案**：引入四种差异化维度，使每张存单卡片具有视觉辨识度。

**差异化维度**：

| 维度 | 实现方式 | 效果 |
|------|---------|------|
| **序号标识** | 卡片左上角显示 "NO.1" "NO.2" 编号 | 快速定位 |
| **进度色阶** | 左色条颜色随进度从浅蓝渐变到深蓝 | 一眼判断存期阶段 |
| **交替底色** | 偶数卡片用极淡交替色 `surfaceVariant(alpha=0.3f)` | 视觉分组 |
| **金额层级** | 大额存单（>50万）用稍大字号 + bold | 突出重点 |

**进度色阶映射**：
```
progress 0-25%   → #7B9FC5（浅蓝，刚开始）
progress 25-50%  → #5C87B0（中蓝）
progress 50-75%  → #4A6FA5（标准蓝 = FinancialColors.deposit）
progress 75-100% → #2E5090（深蓝，快到期了）
```

**卡片布局 v20**：
```
┌──────────────────────────────────┐
│ NO.1  招商银行定期               │  ← 序号 + 名称
│ 本金 100,000 · 年利率 3.50%      │  ← 金额/利率
│ ─────────────────────────────── │
│ 当前估值          今日利息        │
│ 102,500            9.59          │  ← 估值 + 利息（颜色）
│ 2024-01-15 → 2025-01-15         │  ← 日期
│ 480/730 天  ·  累计利息 2,500    │  ← 进度
│ ████████████░░░░░░░░ 65%         │  ← 进度条
│                   [编辑] [删除]  │
└──────────────────────────────────┘
```
