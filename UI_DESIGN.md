# Financial Freedom — UI 高级感重设计方案

> 基于 PRD v15 章节，目标：Apple 级别精致感 × 金融专业信任感

---

## 一、设计理念

### 核心关键词

**克制 · 层次 · 流动 · 金色点缀**

- **克制**：减少颜色数量，信息层级分明，留白充足
- **层次**：白色卡片 + 阴影叠在暖灰背景上，深度感清晰
- **流动**：微动画（数字过渡、涟漪、fade+slide 页面切换）
- **金色点缀**：少量金色（CTA / 选中态 / 分割线）暗示财富属性，不喧宾夺主

### 对标参考

| 维度 | 参考 |
|------|------|
| 卡片质感 | Apple Wallet / 健康 App |
| 信息层级 | Robinhood / 富途牛牛 |
| 色彩控制 | Notion（少色、大量留白） |
| 字体选择 | 高端杂志排版（Vogue / Monocle） |

## 二、字体系统

| 角色 | 字体 | 字重 | 字号 | 行高 |
|------|------|------|------|------|
| Hero 数字（总资产） | Bodoni Moda | 700 | 36sp | 44sp |
| 大标题 | Bodoni Moda | 600 | 24sp | 32sp |
| 卡片标题 | Jost | 600 | 16sp | 24sp |
| 正文 | Jost | 400 | 14sp | 20sp |
| 辅助文本 / 标签 | Jost | 400 | 12sp | 16sp |
| 数字（金额、百分比） | Jost | 500-600 | 16-22sp | — |

> **为什么不用系统默认字体**：Bodoni Moda 的优雅衬线 + Jost 的几何现代感形成「经典 × 当代」对比，传递高级品牌感。这两个字体通过 Google Fonts 分发的 `downloadableFonts` 集成，无需打包字体文件。

### 字重使用规则

```
ExtraBold (700) → 仅 Hero 数字（总资产、今日收益主数字）
Bold (600)     → 分类卡片金额、页面标题
Medium (500)   → 次要数字、按钮文字
Regular (400)  → 正文、标签、辅助信息
```

## 三、色彩系统

### 3.1 浅色模式（默认）

```
页面背景   #F5F5F7  — 微暖浅灰（不是纯白，更有温度）
卡片表面   #FFFFFF  — 纯白浮在暖灰上，层次清晰
主文字     #1C1917  — 近黑（warm black，比 #000 更高级）
次要文字   #6B6560  — 暖灰棕，比常规 gray 更柔和
金色强调   #B7930A  — 选中的指示线 / CTA 按钮 / 分割装饰
金色浅底   #FEF9E7  — 极少数强调背景（如选中日期的底色）
上涨       #C0392B  — 稍暗的正红，不刺眼
上涨浅底   #FDF2F2  — 日历格子 / 标签背景
下跌       #27AE60  — 稍暗的翠绿，不荧光
下跌浅底   #EDF7F0  — 日历格子 / 标签背景
卡片阴影   #000000 · 3% ~ 8% opacity
分割线     #F0EFEC  — 比卡片边框更柔和
```

### 3.2 深色模式

```
页面背景   #0F172A  — 深海蓝黑
卡片表面   #1E293B  — slate 深灰
主文字     #F8FAFC  — 近白
次要文字   #94A3B8  — slate-400
金色强调   #F59E0B  — 深色下更亮
上涨       #EF4444  — 深色下需要更亮
下跌       #4ADE80  — 深色下需要更亮
```

### 3.3 资产分类色（仅用于图标和点缀）

| 资产类型 | 色值 | 用途 |
|----------|------|------|
| 存款 | `#4A6FA5`（稳重的蓝） | 图标、占比条 |
| 股票 | `#C0392B`（涨红色，保持一致） | 图标、占比条 |
| 基金 | `#8B5CF6`（紫色，trust） | 图标、占比条 |
| 黄金 | `#B7930A`（金色） | 图标、占比条 |

> **原则**：分类色只用在图标和细条上，不铺满整个卡片。卡片本身永远是白色。

## 四、间距系统

采用 4dp 基础网格：

| Token | 值 | 用途 |
|-------|-----|------|
| `xs` | 4dp | 图标与文字间距 |
| `sm` | 8dp | 同类信息内部间距 |
| `md` | 12dp | 卡片之间间距 |
| `lg` | 16dp | 卡片内边距 |
| `xl` | 20dp | 大卡片内边距、相关联卡片间距 |
| `2xl` | 24dp | Section 间距 |
| `3xl` | 32dp | 页面顶部/底部留白 |

### 6.2 信息层级（重要性递减）

```
L0 — 今日收益（22sp Bold + 涨跌色，页面视觉锚点）
L1 — 总资产（36sp ExtraBold，用户最关心的数字）
L2 — 走势图（趋势 > 绝对值）
L3 — 资产配置卡片（分类明细）
L4 — 更新时间（10sp，不抢注意力）
```

### 6.3 时间切换器

不用 FilterChip（粗边框丑），改用纯文字 + 金色下划线指示器：

```
        7天         30天        1年
       ────
    (金色短横线在选中项下方)
```

- 选中：`#1C1917` + SemiBold
- 未选中：`#6B6560` + Regular
- 指示线：金色 `#B7930A`，2dp 高，比文字宽 4dp
- 切换动画：`animateColorAsState` + 指示线位置 `animateFloatAsState`

### 6.4 资产占比迷你条

每个分类卡片底部加一条 4dp 高的水平条，金色填充表示占总资产比例：

```kotlin
// 每个 AssetCard 内部
LinearProgressIndicator(
    progress = { assetValue / totalValue },
    color = assetTypeColor,
    trackColor = Color(0xFFF0EFEC),
    modifier = Modifier.height(4.dp).clip(RoundedCornerShape(2.dp))
)
```

## 九、动画规范

| 场景 | 动画 | 时长 |
|------|------|------|
| 数字变化 | `animateIntAsState` / `animateFloatAsState` | 500ms, easeOut |
| 卡片进入 | `AnimatedVisibility` + verticalSlide + fade | 400ms, easeOut |
| 页面切换 | fade(200ms) + horizontalSlide(300ms) | 300ms |
| 卡片点击 | `animateFloatAsState` scale 1.0 → 0.97 → 1.0 | 150ms, spring |
| Tab/时间切换指示线 | `animateFloatAsState` 位置过渡 | 250ms, easeInOut |
| 下拉刷新 | Material 3 内置 PullToRefresh | 默认 |
| 数字滚动 | 使用 `animateFloatAsState` 模拟数字递增 | 600ms, easeOut |
| 涟漪 | Material 3 默认 `ripple()` | 默认 |

> 所有动画需检查 `prefers-reduced-motion`，若开启则 duration = 0。

## 十、图标规范

- 不再使用 emoji（🏦📊💰🥇 → 全部替换）
- 使用 Material Icons Extended：

| 原来 | 替换为 |
|------|--------|
| 💰 存款 | `AccountBalance` |
| 📈 股票 | `TrendingUp` |
| 💵 基金 | `WaterDrop`（水滴 = 流动 = 资金） |
| 🥇 黄金 | `Diamond` |

- 图标大小：24dp 统一 `Modifier.size(24.dp)`
- 导航栏：Outlined（未选中）/ Filled（选中）
- 所有图标使用 `MaterialTheme.colorScheme.onSurface`

## 十一、实现要点

### 11.1 Theme.kt

- 新增自定义 `ColorScheme`，不依赖 `dynamicColor`（Material You 取色不可控，无法保证金色点缀的一致性）
- 浅色/深色两套 `ColorScheme`，手动定义
- 新增 `FinancialTypography` 使用 Bodoni Moda + Jost

### 11.2 字体下载

```kotlin

val JostFont = GoogleFont("Jost", FontWeight.Normal)
```
